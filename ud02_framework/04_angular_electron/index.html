    Introducción a Angular

    Angular es un framework de desarrollo web de código abierto mantenido por Google. Está diseñado para facilitar la creación de aplicaciones web dinámicas y de una sola página (SPA) mediante el uso de componentes reutilizables, inyección de dependencias y un sistema de enlace de datos (data binding) bidireccional.

    Principales Características:

    • TypeScript: Utiliza TypeScript, un superconjunto de JavaScript que añade tipado estático y características avanzadas de programación orientada a objetos.
    • Estructura: Proporciona una estructura clara y organizada para el desarrollo de aplicaciones, lo que facilita la escalabilidad y el mantenimiento del código.
    • Componentes: Arquitectura basada en componentes, donde cada componente representa una parte específica de la interfaz de usuario y su lógica asociada.
    • Plantillas: Utiliza plantillas HTML para definir la estructura de la interfaz de usuario, junto con directivas y data binding para interactuar con el modelo de datos.
    • Data Binding Bidireccional: Permite la sincronización automática entre el modelo de datos y la vista, lo que simplifica la actualización de la interfaz de usuario.
    • Directivas: Proporciona directivas que permiten manipular el DOM de manera declarativa, facilitando la creación de componentes personalizados y la reutilización de código.
    • Servicios: Utiliza servicios para encapsular la lógica de negocio y compartir datos entre componentes.
    • Routing: Incluye un sistema de routing que permite la navegación entre diferentes vistas y componentes dentro de una aplicación de una sola página.
    • Inyección de Dependencias: Facilita la gestión de dependencias mediante un sistema de inyección de dependencias, lo que mejora la modularidad y la reutilización del código.
    • CLI (Command Line Interface): Angular CLI es una herramienta de línea de comandos que facilita la creación, desarrollo y despliegue de aplicaciones Angular.
    • Formularios: Proporciona soporte robusto para la creación y validación de formularios, tanto reactivos como basados en plantillas.
    • Testing: Incluye herramientas integradas para pruebas unitarias y de extremo a extremo, lo que facilita la garantía de calidad del código.
    • Internacionalización: Soporta la creación de aplicaciones multilingües mediante herramientas de internacionalización.

    Creación de una Aplicación Angular

    En este curso utilizaremos la última versión estable de Angular (en el momento de escribir este documento, la versión es la 20).

    Node.js

    En la documentación oficial de Angular se indican los requisitos previos necesarios para trabajar con Angular https://angular.dev/reference/versions.

    Para Angular 20 se debe instalar la ultima versión estable (LTS) de Node.js (22.19.0 en el momento de escribir este documento).

    Se recomienda el uso de nvm (Node Version Manager) para gestionar las versiones de Node.js.

    Instalación de Angular CLI

    Desde un terminal de línea de comandos, ejecuta el siguiente comando para instalar la interfaz de línea de comandos de Angular (Angular CLI):

    npm install -g @angular/cli

    Comprobar la instalación:

    Con el comando ng version se puede comprobar la versión instalada de Angular CLI y otras dependencias relacionadas.

    Comprobar la versión instalada
    ng version

    Integración de Angular con Electron

    Como vimos en el apartado anterior, Electron permite crear aplicaciones de escritorio utilizando tecnologías web como HTML, CSS y JavaScript. Al combinar Electron con Angular, podemos aprovechar las capacidades de ambos para desarrollar aplicaciones de escritorio.

    Pasos para integrar Angular con Electron

    1. Crear una aplicación Angular

      Si no tienes una aplicación Angular existente, crea una nueva utilizando el comando ng new my-angular-app.

      Nota

      Si ya tienes una aplicación Angular, puedes omitir este paso.

    2. Instalar Electron

      Navega al directorio de tu proyecto y ejecuta el siguiente comando para instalar Electron como una dependencia de desarrollo:

      npm install electron --save-dev

    3. Configurar el archivo principal de Electron

      Crea un archivo llamado main.js en la raíz de tu proyecto con el siguiente contenido:

      main.js
      const { app, BrowserWindow, screen } = require('electron');
const path = require('path');

process.on('uncaughtException', (error) => {
    console.error('Uncaught Exception:', error);
});

function createWindow() {
    const size = screen.getPrimaryDisplay().workAreaSize;

    const win = new BrowserWindow({
        webPreferences: {
            width: 800,
            height: 600,
            contextIsolation: true,
            nodeIntegration: false,
        }
    });

    // Cargar la aplicación Angular dependiendo del entorno

    if (!app.isPackaged) {

        // En desarrollo, cargar desde el servidor de desarrollo de Angular

        win.loadURL('http://localhost:4200');
        win.webContents.openDevTools();

    } else {

        // En producción, cargar el archivo index.html generado por Angular

        const indexPath = path.join(__dirname, 'dist', 'my-angular-app', 'browser', 'index.html');
        console.log('Loading file:', indexPath);
        win.loadFile(indexPath);

        win.webContents.on('did-fail-load', () => {
            // Reintentar cargar el archivo en caso de fallo, esto previene errores en algunas plataformas
            console.log('Retrying to load file:', indexPath);
            win.loadFile(indexPath);
        });
    }

    // Maximizar la ventana y mostrarla
    win.maximize();
    win.show();
}

app.whenReady().then(createWindow);

app.on('window-all-closed', () => {
    if (process.platform !== 'darwin') app.quit();
});

app.on('activate', () => {
    if (BrowserWindow.getAllWindows().length === 0) createWindow();
});

    4. Instalar dependencias adicionales

      Para facilitar la gestión de tareas y la construcción de la aplicación, instala las siguientes dependencias adicionales:

      Instalación de dependencias adicionales
      npm install electron-builder wait-on npm-run-all --save-dev
      • electron-builder: Herramienta para empaquetar y distribuir aplicaciones Electron.
      • wait-on: Utilidad para esperar a que un recurso esté disponible (útil para esperar a que el servidor de desarrollo de Angular esté listo).
      • npm-run-all: Herramienta para ejecutar múltiples scripts de npm en paralelo o secuencialmente.

    7. Ejecutar la aplicación en modo desarrollo

      Utiliza el siguiente comando para iniciar la aplicación en modo desarrollo. Esto iniciará tanto el servidor de desarrollo de Angular como la aplicación Electron.

      npm run dev

      Este modo es el que utilizaremos durante el desarrollo de la aplicación, ya que permite ver los cambios en tiempo real sin necesidad de reconstruir la aplicación, a no ser que se modifique el archivo main.js, en cuyo caso será necesario reiniciar la aplicación Electron.

    8. Crear una versión empaquetada

      Para crear una versión empaquetada de la aplicación, utiliza uno de los siguientes comandos:

      • Para crear un paquete para la plataforma actual (Windows, Linux o macOS):
      npm run package
      • Para crear un paquete específico para Windows:
      npm run package-win
      • Para crear un paquete específico para Linux:
      npm run package-linux

      Los archivos empaquetados se encontrarán en el directorio dist/ dentro del proyecto.

    Errores comunes

    • El ejecutables no se inician o la ventana aparece en blanco.

      Asegúrate de que la ruta al archivo index.html en el archivo main.js es correcta. La ruta debe coincidir con la estructura de tu proyecto y el nombre de la aplicación Angular.

      • Dentro del archivo main.js, la línea que carga el archivo index.html apunte al lugar correcto:
      const indexPath = path.join(__dirname, 'dist', 'my-angular-app', 'browser', 'index.html');

      Mira el directorio dist/ para verificar que la estructura de carpetas y el nombre de la aplicación coinciden.

      • Asegúrate de que has ejecutado ng build para construir la aplicación Angular antes de empaquetar con Electron.
      • Comprueba la configuración del archivo package.json, en particular la sección build.files, para asegurarte de que incluye los archivos necesarios:
      package.json: fragmento de build.files
      "build": {
  "files": [
      "dist/my-angular-app/**/*",
      "main.js",
      "package.json"
  ]
