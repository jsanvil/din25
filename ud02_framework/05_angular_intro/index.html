    En la sección anterior vimos cómo configurar el entorno de desarrollo, creando una aplicación básica de Electron con Angular. En esta sección, profundizaremos en los conceptos fundamentales de Angular para empezar a desarrollar interfaces de usuario.

    Estructura de un proyecto Angular

    Para explicar los conceptos básicos de Angular, utilizaremos un proyecto de ejemplo llamado angular-productsp que iremos creando paso a paso.

    Crear proyecto Angular
    ng new angular-products

    Una vez creado el proyecto, su estructura de directorios y archivos es la siguiente:

    Estructura inicial del proyecto
    angular-products/
├── .git/
├── .vscode/
├── node_modules/
├── public/
├── src/
│   ├── app/
│   │   ├── app.config.ts
│   │   ├── app.routes.ts
│   │   ├── app.css
│   │   ├── app.html
│   │   ├── app.spec.ts
│   │   └── app.ts
│   ├── index.html
│   ├── main.ts
│   └── styles.css
├── .editorconfig
├── .gitignore
├── angular.json
├── package.json
├── package-lock.json
├── README.md
├── tsconfig.app.json
├── tsconfig.json
└── tsconfig.spec.json

    En esta estructura, los archivos y carpetas más importantes son:

    • src/: Contiene el código fuente de la aplicación.
      • app/: Aquí es donde se encuentran los componentes, servicios y módulos de la aplicación.
        • app.ts: Archivo principal del módulo de la aplicación.
        • app.html: Plantilla HTML principal de la aplicación.
        • app.css: Estilos CSS principales de la aplicación.
        • app.routes.ts: Define las rutas de navegación de la aplicación.
        • app.config.ts: Configuraciones globales de la aplicación.
      • index.html: Archivo HTML principal que carga la aplicación Angular.
      • main.ts: Punto de entrada de la aplicación Angular.
      • styles.css: Archivo global de estilos CSS.
    • angular.json: Archivo de configuración del proyecto Angular.
    • package.json: Archivo que define las dependencias y scripts del proyecto.
    • README.md: Punto de entrada para la documentación del proyecto.
    • tsconfig.json: Configuración del compilador TypeScript.
    • *.spec.ts: Archivos de pruebas unitarias.

    Componentes

    Los componentes son las unidades básicas de una aplicación Angular. Cada componente consta de tres partes principales:

    1. Clase TypeScript: Define la lógica del componente.
    2. Plantilla HTML: Define la estructura visual del componente.
    3. Estilos CSS: Define la apariencia del componente.

    Referencia: https://angular.dev/essentials/components

    Para crear un nuevo componente, podemos usar el siguiente comando:

    Crear componente
    ng generate component <nombre-del-componente>

    O en su forma abreviada:

    Crear componente
    ng g c <nombre-del-componente>

    Como ejemplo, para crear un componente llamado hello, que mostrará un mensaje de saludo, usaríamos:

    Crear componente Hello
    ng g c hello

    Esto generará una carpeta hello/ dentro de src/app/ con los siguientes archivos:

    Estructura del componente Hello
    src
├── app
│   ├── app.config.ts
│   ├── app.css
│   ├── app.html
│   ├── app.routes.ts
│   ├── app.spec.ts
│   ├── app.ts
│   └── hello
│       ├── hello.css
│       ├── hello.html
│       ├── hello.spec.ts
│       └── hello.ts
├── index.html
├── main.ts
└── styles.css

@Component({
  selector: 'app-hello',

    CSS

    Se ve una separación clara entre la lógica (TypeScript), la estructura (HTML) y el estilo (CSS), puede que parezca complejo al principio, teniendo en cuenta un ejemplo tan simple, pero esta separación es fundamental para mantener el código organizado y facilitar su mantenimiento.

    Interpolación de variables {{ }}

    Para crear un enlace entre el valor de una variable en TypeScript y representarlo en la plantilla HTML se utiliza las dobles llaves {{ variable }}

    De esta forma, podemos crear valores dinámicos en html, controlando su valor desde el código del componente.

    Referencia: https://angular.dev/guide/templates/binding#render-dynamic-text-with-text-interpolation

    Para utilizar este componente en la aplicación, debemos declararlo en el módulo principal (app.ts) y luego incluir su selector (<app-hello>) en la plantilla HTML principal (app.html).

Y Si necesitamos cambiar la lógica, la estructura o el estilo del componente, podemos hacerlo de manera independiente sin afectar otras partes de la aplicación.

    Librerías de estilos

    Antes de continuar añdiendo más componentes, vamos a mejorar la apariencia de nuestra aplicación utilizando una librería de estilos. En este caso, usaremos Bootstrap, que es una de las librerías CSS más populares y fáciles de usar.

    Para instalar Bootstrap, ejecuta el siguiente comando en la terminal:

    Instalar Bootstrap
    npm install bootstrap

    Para configurarlo como estilo global podríamos hacerlo de varias formas:

    • Importarlo directamente en el archivo styles.css
    • Añadirlo en la sección de estilos del archivo angular.json

    Importante

    No es necesario hacer las dos importaciones, con una es suficiente.

    Importar Bootstrap en styles.css

    Directivas @if y @for

    El siguiente paso es añadir productos al listado, y el primer paso será especificar la estructura que tendrán los datos mediante una interfaz que llamaremos Product

    Generar nuevo interface Product
    ng generate interface interfaces/product

    Generar elementos en directorios específicos

    Como se puede observar en el ejemplo, hemos especificado la ruta donde se generará el nuevo interface interfaces/product

    Es importante mantener una buena organización en los proyectos, para facilitar la localización de los múltiples archivos fuente que lo componen.

    Creación de interfaces en Angular

    Para generar una nueva interfaz utilizaremos el comando ng de la siguiente forma:

    ng generate interface <nombre-nueva-interfaz>

    O en su forma abreviada:

    ng g i <nombre-nueva-interfaz>

    Ahora debemos definir la nueva interfaz estableciendo sus propiedades y tipos de datos:

    import { Component } from '@angular/core';
import { MatCardModule } from '@angular/material/card';
import { Product } from '../interfaces/product';

@Component({
  selector: 'app-product-list',
  imports: [MatCardModule],
  templateUrl: './product-list.html',

export class ProductList {
  title = 'Mi lista de productos';
  headers = {
    description: 'Producto',
    price: 'Precio',
    available: 'Disponible'
  }

  products: Product[] = [
    {

      available: '2024-09-15',

  ];
}

    @if(products && products.length) {
    <div class="table-responsive">
      <table class="table table-striped">
        <thead>
          <tr>
            <th>{{headers.description}}</th>
            <th>{{headers.price}}</th>
            <th>{{headers.available}}</th>
          </tr>
        </thead>
        <tbody>
          @for(product of products; track product.id) {
          <tr>
            <td>{{product.description}}</td>
            <td>{{product.price}}</td>
            <td>{{product.available}}</td>
          </tr>
          }
        </tbody>
      </table>
    </div>
    }
    @else {
    <div class="alert alert-info">
      No hay productos
    </div>
    }
  </mat-card-content>
</mat-card>

    Resultado de lista con productos:

    Resultado de lista vacía:

    La directiva @for tiene una propiedad track obligatoria, que permite especificar una propiedad del objeto que se utiliza para identificar cada elemento de la lista. Esto permite que Angular pueda identificar los elementos de la lista y actualizarlos de forma eficiente.

    @empty

    @for se puede acompañar de @empty, por ejemplo:

    @for (item of items; track item.name) {
    <li> {{ item.name }}</li>
} @empty {
    <li> Ho hay elementos. </li>
}

    En el ejemplo visto con el listado de productos no se utiliza, porque, cuando no haya elementos, se quiere ocultar toda la tabla, incluyendo cabeceras.
