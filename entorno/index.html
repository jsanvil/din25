    🔧 Entorno de trabajo

    En este curso utilizaremos las siguientes tecnologías:

    • Repositorios de Git alojados en GitHub para el seguimiento y entrega de las prácticas y proyectos.
    • Visual Studio Code como editor de código.
      • Alternativamente se puede utilizar VSCodium que es una versión de código abierto de VSCode.
    • Node.js como entorno de ejecución.
      • nvm para la gestión de versiones de Node.js en el aula de informática.
    • Electron para el desarrollo de aplicaciones de escritorio multiplataforma.
    • Angular para el desarrollo de interfaces gráficas.
    • TypeScript como lenguaje de programación principal.

    Visual Studio Code

    Para la instalación de VSCode, descargar el paquete de instalación que corresponda al sistema operativo de la página de Visual Studio Code y realizar la instalación típica.

    Versión portable

    Si no se dispone de permisos de administrador para instalar software, se puede utilizar la versión portable de VSCode.

    En la página oficial de descargas disponemos de empaquetados .zip para MS Windows y .tar.gz para Linux.

    Sólo hay que descargar el archivo, descomprimir y ejecutar el archivo code.

    Node.js npm y nvm

    npm es el gestor de paquetes de Node.js. Para instalarlo se debe seguir la guía en la página de nodejs.org.

    Instalación en el perfil local de usuario para sistemas Linux

    Alternativamente, npm se puede instalar en el perfil local del usuario, para ello utilizaremos nvm que es un gestor de versiones de node que nos permitirá utilizar la versión que escojamos nvm-sh

    Para instalar nvm necesitamos ejecutar el siguiente comando, aunque es recomendable consultar la documentación de nvm para comprobar que no haya cambiado:

    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash

    wget

    Si no se dispone de curl, se puede utilizar wget:

    wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash

    Cuando termine la instalación de nvm, tendremos que abrir un nuevo terminal para que reconozca la ruta del ejecutable y proceder a instalar la última versión LTS de Node.js de la siguiente forma:

    nvm install --lts

    Git, GitHub y GitHub Classroom

    • Git es un sistema de control de versiones usado comúnmente para código fuente.
    • GitHub es una plataforma de desarrollo colaborativo que utiliza Git y que nos permitirá subir nuestro código a la nube.
    • GitHub Classroom es una herramienta que nos permitirá recibir las tareas y ejercicios de la asignatura.

    Por lo tanto será necesario:

    • Si no tenemos cuenta en GitHub, Crear una cuenta de GitHub utilizando la cuenta de correo corporativa de la GVA (usuario@alu.edu.gva).

      • Utiliza el mismo nombre de usuario para la cuenta GitHub.

      Ejemplo: Para la cuenta corporativa es jacsanvil@alu.edu.gva el usuario de GitHub debe ser jacsanvil

    • Instalar Git

      Si no estás familiarizado en cómo funciona git, se recomienda leer la Guía de Git.

    • Para tareas sencillas, se recomienda el gestor integrado de Visual Studio Code.

    • Para tareas más complejas, se recomienda utilizar la herramienta de línea de comandos de Git o un cliente gráfico como GitHub Desktop.
